# Landing Page Optimization PRD — keratocones.com

**Synthesized from 3 independent research PRDs (A: Evidence-Only, B: Mobile Conversion, C: Quality Score)**
**Cold Review:** Completed — 1 CRITICAL + 3 HIGH issues found and fixed below
**Date:** 2026-03-26
**For:** Claude working in keratocones_website project
**Research basis:** 28+ cited sources, 15,000+ account QS study, Unbounce benchmark, PubMed KC patient study

## ⚠️ COLD REVIEW FIXES (Applied 2026-03-26)

### CRITICAL FIX: Mobile form must have 3 fields, not 2
The API route at `app/api/contact/route.ts` requires firstName, lastName, phone, AND email.
A 2-field form (name + phone) will return 400 error. Mobile form must be:
- **Name** (single field — split into first/last server-side)
- **Phone** (required)
- **Email** (required — GHL uses email for dedup)
- Submit button

### HIGH FIX: Pick ONE sticky element, not two
Sticky nav bar + sticky CTA bar = 15-17% of mobile viewport permanently consumed.
**Decision: Use sticky CTA bar ONLY** (bottom, call + book). Drop the sticky nav bar.
In-page navigation can be a scrollable horizontal bar that scrolls with content, NOT sticky.

### HIGH FIX: Hero text comes from geo-personalization
The hero content is DYNAMIC — generated by `lib/geo-personalization.ts`, not hardcoded in `Hero.tsx`.
DO NOT hardcode "Orange County's leading keratoconus specialist" in the component.
Instead: add "near me" and location keywords to the GEO-PERSONALIZATION templates in `lib/geo-personalization.ts`.

### HIGH FIX: Verify actual component names
- The page uses `LeadForm` component (wraps `ContactForm`) — edit BOTH
- Form section has `id="lead-form"` in page.tsx, inner div has `id="contact-form"` in LeadForm.tsx
- `MidPageCTA` component appears twice — keep these, they work alongside sticky CTA
- `FadeIn` wraps most sections — fix uses CSS `opacity: 1` default overridden by Framer Motion on hydration

### MEDIUM FIX: Address cost anxiety without showing price range
Don't show "$1,500-$4,500" but DO add: "Most insurance plans cover keratoconus treatment. We verify your benefits before your visit." This addresses the #1 patient concern without anchoring a scary number.

### MEDIUM FIX: Accordion should use CSS visibility, not DOM duplication
Use a single element with `max-height: 0; overflow: hidden` on mobile, not two separate DOM elements. Avoids content duplication and extra page weight.

### ITEMS RESTORED FROM SOURCE PRDs
- Add CHOC + UCI referral logos near doctor photo (PRD-A)
- Add "What to Expect at Your First Visit" section (PRD-C)
- Add inline CTA within FAQ section (PRD-A)
- Add `MedicalProcedure` schema for scleral lens fitting (PRD-C)
- Acceptance criterion #13: form submits successfully end-to-end with GHL API after changes

## The Problem

keratocones.com converts Google Ads clicks at ~2-3%. Healthcare median is 5.1-7.4%. Our Quality Score is 2-4 on key keywords (should be 7+). Fixing QS alone would cut CPC by 36% = 56% more clicks at the same budget.

**The page has all the right content — it's in the wrong ORDER.**

93% of clicks are mobile. The contact form is 17-19 thumb-scrolls deep (section 14 of 14). Dr. Bonakdar's photo is at 75% scroll depth. The page was designed as a desktop content site, not a mobile conversion funnel.

## Changes (Priority Order)

### PRIORITY 1: Quality Score Fixes (Free CPC reduction)

#### 1.1 Fix page load speed
**File:** `app/page.tsx`, `app/layout.tsx`, `next.config.ts`

The page loads 98 JS bundles, a YouTube iframe, Facebook Pixel, and multiple Cloudinary images on initial render. Target: LCP < 2.5s mobile.

Changes:
- Lazy-load YouTube video (load iframe only when section scrolls into view)
- Defer Facebook Pixel and LinkedIn Insight tag (load after `onLoad`)
- Add `loading="lazy"` to all images below the fold
- Code-split non-critical components (FAQ accordion, insurance section, city silos)
- Consider replacing hero background image with a smaller optimized version or CSS gradient on mobile

#### 1.2 Fix FadeIn component for AdsBot
**File:** `components/FadeIn.tsx`

The `FadeIn` animation component likely sets `opacity: 0` on initial render, then animates to `opacity: 1`. Google's AdsBot-Google-Mobile does NOT reliably execute JavaScript animations. Content wrapped in FadeIn may be invisible to Google's QS crawler.

Fix: Add `<noscript>` fallback OR set initial CSS to `opacity: 1` with a class that triggers animation only after JS hydration:
```tsx
// Instead of starting at opacity: 0
// Start at opacity: 1, then add animation class after mount
const [mounted, setMounted] = useState(false);
useEffect(() => setMounted(true), []);
// Only apply fade animation after mounted
```

#### 1.3 Add missing keyword phrases to page copy
**File:** `components/Hero.tsx`, `components/KeratoconusContent.tsx`

Current page has 0 occurrences of "near me", "orange county keratoconus specialist", or "cornea specialist." These are bidded keywords with low QS.

Add naturally (not keyword-stuffed):
- Hero subhead: "Orange County's leading keratoconus specialist — 5,000+ cases, same-week appointments, direct specialist access **near you**."
- About section: "Dr. Bonakdar is a **keratoconus specialist near** Santa Ana, serving patients across **Orange County** and Southern California."
- Add city mention: "Located in Santa Ana, minutes from Irvine, Costa Mesa, Tustin, and Newport Beach."

#### 1.4 Add in-page navigation
**File:** `app/page.tsx` or new component `components/StickyNav.tsx`

Google's February 2025 navigation prediction model penalizes pages where users cannot "efficiently access desired information." The page has 14 sections with zero in-page navigation.

Add a sticky anchor nav bar (appears after scrolling past hero):
```
[Symptoms] [Treatment Options] [About Dr. B] [Insurance] [Book Now]
```
Each links to the corresponding section via `#anchor`. This helps both QS (navigation score) and conversion (users can jump to what they need).

### PRIORITY 2: Mobile Conversion Fixes (More form submissions)

#### 2.1 Add sticky mobile CTA bar
**File:** New component `components/StickyCTA.tsx`, add to `app/layout.tsx`

A persistent bottom bar on mobile (hidden on desktop) with two buttons:
```
[📞 Call Now]  [📋 Book Online]
```
- "Call Now" → `tel:+17145580641`
- "Book Online" → scrolls to form OR opens popup form (see 2.3)
- Only visible after scrolling past hero (don't cover hero CTA)
- Height: 56-64px, fixed to bottom of viewport
- Research: sticky CTAs increase mobile clicks by 27%

#### 2.2 Move Dr. Bonakdar's photo + credentials higher
**File:** `app/page.tsx` (section reorder), `components/Hero.tsx`

Currently: Doctor photo is in "About This Practice" section at ~75% scroll depth.
Move to: Just below the hero testimonials (section 3), above the VS comparison.

The research is clear: **seeing the doctor's face increases conversions 38-95%** (5 independent sources). Patients want to know WHO they're trusting with their eyes before reading educational content.

Proposed new section order (mobile):
1. Hero (H1 + CTA + phone + badges) — keep as-is
2. Patient testimonials (3 reviews) — keep position
3. **Doctor photo + credentials** ← MOVED UP from section 12
4. VS comparison (University Hospitals vs us) — keep
5. **Sticky contact form** or form trigger ← MOVED UP from section 14
6. Scleral lens assessment quiz
7. Vision slider (before/after)
8. Educational content (collapsible on mobile)
9. FAQ
10. Insurance + CareCredit
11. Full contact form (for desktop / if not using popup)
12. Footer

#### 2.3 Simplify the form (mobile especially)
**File:** `components/ContactForm.tsx`

Current: 5 fields (first name, last name, phone, email, message) + 2 checkboxes = 7 interaction points.
Research: Reducing fields from 5 to 3 = 120% conversion lift (Tebra/PatientGain).

**Mobile form (above fold or popup) — 3 fields:**
- Name (single field — split into first/last server-side in the API route)
- Phone (required)
- Email (required — GHL uses email as dedup key)
- Submit button: "Get a Call Back Today"

**IMPORTANT:** The API route at `app/api/contact/route.ts` line 25 requires firstName, lastName, phone, AND email. If you use a single "Name" field, split it server-side:
```typescript
const [firstName, ...rest] = name.trim().split(' ');
const lastName = rest.join(' ') || firstName; // fallback if single name
```

**Desktop form (full, at bottom):**
- Keep current 5 fields + checkboxes (more screen space, lower friction on desktop)

**Test:** After changes, verify form submission creates a GHL contact. Submit a test → check GHL API → contact exists with correct fields.

#### 2.4 Add "Lenses Before Surgery" section
**File:** New component or section in `app/page.tsx`

KC patients' top concern (PubMed study): "Will I need surgery?" Many are referred by surgeons pushing cross-linking first. Add a section between the VS comparison and educational content:

**Heading:** "Do I Need Surgery for Keratoconus?"
**Content (3-4 sentences):**
"Surgery isn't always the first step. For many keratoconus patients, custom scleral lenses restore clear vision without any surgical procedure. Dr. Bonakdar evaluates each patient individually — in over 5,000 cases, the majority achieved excellent vision with specialty lenses alone. Cross-linking may be recommended to stop progression, but lenses come first for vision restoration."

This directly addresses the #1 patient fear and differentiates from surgeons pushing CXL as the first option.

#### 2.5 Contextualize the 5,000+ number
**File:** `components/Hero.tsx` or testimonials section

Current: "5,000+ cases" (raw number, no context).
Better: "Most eye clinics fit 10 keratoconus patients per year. **Dr. Bonakdar has fitted over 5,000.**"

Add this as a callout or badge near the testimonials section. The context makes the number meaningful — 5,000 alone could mean 5,000 over 50 years (100/year). With context, it shows massive specialization.

### PRIORITY 3: Content Improvements (SEO + Trust)

#### 3.1 Educational content accordion on mobile
**File:** The long-form content sections (symptoms, why glasses don't work, progression, lens options, monitoring)

These 7 sections are ~3,000 words of excellent educational content. On desktop, they work. On mobile, they're a wall of text between the user and the contact form.

Fix: Wrap in `<details>/<summary>` or an accordion component on mobile. Content stays in DOM for SEO but doesn't block the conversion path.

```tsx
// Mobile: collapsible
<details className="md:hidden">
  <summary>Common Symptoms of Keratoconus</summary>
  {/* full content */}
</details>

// Desktop: always visible
<div className="hidden md:block">
  {/* full content */}
</div>
```

#### 3.2 Add structured data
**File:** `app/page.tsx` or `components/BusinessSchema.tsx`

Add `BreadcrumbList` and `SiteNavigationElement` schema. These help Google understand the page navigation structure (QS navigation score).
